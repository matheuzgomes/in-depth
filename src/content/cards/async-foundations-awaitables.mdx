Native coroutines are driven by `await`. The important mental model is not "async means parallel" but "the coroutine can suspend cooperatively so the event loop can run something else."

```python
# Works on Python 3.7+
import asyncio

async def fetch_flag():
    await asyncio.sleep(0.1)
    return "BR"

print(asyncio.run(fetch_flag()))
```

<Callout type="info">
  <code>await</code> builds on the same coroutine-driving ideas that Fluent Python traces back to generators, but now through Python's native coroutine syntax and awaitable protocol.
</Callout>
