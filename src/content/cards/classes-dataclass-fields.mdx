`@dataclass` generates methods from annotations, but field options decide mutability, ordering, defaults, and initialization flow.

```python
# [OLDER / 3.9, CURRENT - 3.10-3.14] Works on Python 3.9+ [PEP 585]
from dataclasses import dataclass, field

@dataclass(frozen=True, order=True)
class Job:
    priority: int
    tags: list[str] = field(default_factory=list)

print(Job(1))
```

<Callout type="warn">
  Use <code>default_factory</code> for mutable field defaults. It solves the same shared-state trap as function defaults.
</Callout>
