Container sequences store references to Python objects. Flat sequences store raw values, which can be much tighter and friendlier to I/O.

```python
# [CURRENT - 3.10-3.14] Works on Python 3.x
view = memoryview(bytearray(b"abcdef"))
view[2:4] = b"XY"  # zero-copy slice assignment
print(bytes(view))
```

<Callout type="info">
  For one-pass data, combine this with generators: flat storage when you must keep values, lazy iteration when you do not.
</Callout>
