Async iteration lets the consumer await each next value instead of demanding the whole dataset synchronously. Async generator functions make that pattern concise and native.

```python
# [OLDER / 3.6-3.9, CURRENT - 3.10-3.14] Requires Python 3.6+ [PEP 525]
import asyncio

async def ticker():
    for value in (1, 2, 3):
        await asyncio.sleep(0.1)
        yield value
```

<Callout type="tip">
  <code>async for</code> is still pull-based iteration. It does not create parallelism by itself; it inserts awaitable boundaries between items.
</Callout>
