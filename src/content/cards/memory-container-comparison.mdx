If the question is “which container should I pick?”, start by separating four jobs: mutable ordered work (`list`), fixed ordered record (`tuple`), repeated membership/uniqueness (`set`), and dense homogeneous numbers (`array.array`).

```python
# [CURRENT - 3.10-3.14] Works on Python 3.x
from array import array

ordered = [10, 20, 30]
record = (10, 20, 30)
unique = {10, 20, 30}
packed = array("I", [10, 20, 30])

print(type(ordered), type(record), type(unique), type(packed))
```

<Callout type="warn">
  Do not compare only syntax. <code>set</code> is a hash table, <code>list</code> and <code>tuple</code> are object-reference sequences, and <code>array.array</code> is packed C storage. Their memory and performance costs come from different representations.
</Callout>
