Sequence patterns let you branch on shape first, then bind names. That keeps protocol parsing away from fragile `parts[0]`, `parts[1]` code.

```python
# Requires Python 3.10+ [PEP 634]
def route(command):
    match command.split():
        case ["move", x, y]:
            return ("move", float(x), float(y))
        case ["stop"]:
            return ("stop",)
        case _:
            raise ValueError(command)

print(route("move 3 4"))
```

<Callout type="warn">
  <code>str</code>, <code>bytes</code>, and <code>bytearray</code> are not matched as sequence patterns. Split or parse them first.
</Callout>
