Python calls them **special method names**. The community nickname is **dunder methods** because they use double underscores on both sides, like `__eq__` and `__iter__`.

```python
# [CURRENT - 3.10-3.14] Works on Python 3.x
class UserId:
    def __init__(self, value):
        self.value = value

    def __eq__(self, other):
        if not isinstance(other, UserId):
            return NotImplemented
        return self.value == other.value

print(UserId(7) == UserId(7))
```

<Callout type="warn">
  <code>==</code> may dispatch to <code>__eq__</code>. <code>is</code> never does. Identity is a direct object check, not an overload point.
</Callout>
