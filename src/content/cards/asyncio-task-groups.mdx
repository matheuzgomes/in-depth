`asyncio` concurrency is cooperative. Tasks switch only at `await`, so blocking work blocks every task on the event loop.

```python
# Requires Python 3.11+
import asyncio

async def fetch_user():
    await asyncio.sleep(0.1)
    return {"id": 1}

async def fetch_orders():
    await asyncio.sleep(0.1)
    return ["A-100", "A-101"]

async def main():
    async with asyncio.TaskGroup() as tg:
        user_task = tg.create_task(fetch_user())
        orders_task = tg.create_task(fetch_orders())
    return user_task.result(), orders_task.result()

print(asyncio.run(main()))
```

<Callout type="tip">
  Use semaphores or queues to bound fan-out. Unbounded <code>gather()</code> can create more concurrent work than the system can absorb.
</Callout>
