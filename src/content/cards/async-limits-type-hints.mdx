Async helps when work waits on external readiness. It does not make CPU-bound Python code parallel, and it does not erase the need to type protocols like `Awaitable`, `AsyncIterator`, or `AsyncGenerator` correctly.

```python
# [OLDER / 3.9, CURRENT - 3.10-3.14] Works on Python 3.9+ [PEP 585]
from collections.abc import AsyncIterator

async def stream_ids() -> AsyncIterator[int]:
    for value in (1, 2, 3):
        yield value
```

<Callout type="warn">
  If a coroutine spends a long time in CPU work without awaiting, the event loop cannot schedule anything else. Async syntax does not fix that.
</Callout>
